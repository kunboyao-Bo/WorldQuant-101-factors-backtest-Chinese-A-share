# WorldQuant 101 Factors Backtest — Chinese A-Share

基于 WorldQuant 101 Formulaic Alphas 论文，针对中国 A 股市场的因子有效性验证与多因子选股框架实现。

------------------------------------------------------

## 项目简介

本项目旨在将 WorldQuant 提出的 101 个量价 Alpha 因子移植到 A 股市场，通过 Portfolio Sort 对因子有效性进行检验

数据来源：
量价数据、公司ST信息 -> tushare
marketcap、行业信息 -> LSEG api
------------------------------------------------------

## 范围及条件
1.时间范围选取2022-2025所有A股交易日
2.股票容量为全量A股上市公式
3.量价数据包括close open high low volume amount adj_factor（前复权因子） 数据格式为 DataFrame -> (index = date, columns = stock_id)
    其中 Tushare 单位：amount 千元，vol 手（100股）vwap = （amount* 1000） / (vol * 100)
4.财务信息包括股票行业、ST信息、市值
5.换仓频率为周

------------------------------------------------------

## 回测标准

1.IC均值绝对值 > 0.02 
2.ICIR绝对值 > 0.3    
3.IC t统计量显著, 同时要求样本量足够（>50期）
4.Q5-Q1 正且稳定,t统计量显著
5.分组严格单调性
------------------------------------------------------

```
## 回测方法


    ```
    数据加载 → 前复权处理 → 逐因子循环
        ├── run_mode=1：IC Decay 分析（确认换仓频率）目前默认是周频率
        └── run_mode=2：Portfolio Sorting 分组回测 → IC计算 → Excel报告输出
    ```

    ---

    一、数据预处理

    **前复权（qfq）**

    原始行情数据为未复权价格，统一做前复权处理：
    复权价 = 原始价 × adj_factor / 最新adj_factor
    以最新一期复权因子为基准，保证当前价格不变、历史价格向前调整。close / open / high / low / vwap 均做复权，volume / amount 保持原始值不变。

    **VWAP 计算**

    vwap = amount(元) × 1000 / (volume(手) × 100)
         = 每股成交均价（元/股）

    ---

    二、股票池过滤 `market_cap_ST_filter`

    每个换仓日动态过滤，逻辑如下：

    1. 取信号日之前最近一个有数据的月末市值截面
    2. 按市值升序排列，保留**后70%**（即剔除市值最小的30%）
    3. 从 ST 信息表中提取所有含"ST"字样的股票代码，从股票池中剔除
    返回当期有效股票列表。

    ---

    三、IC Decay 分析 `calc_ic_decay`（run_mode=1）

    **目的**：在正式回测之前，先检验因子的预测能力随持有期延长的衰减速度，用于决定合理换仓频率。

    **逻辑：**

    对每个 lag（1 到 max_lag 天）：
    1. 计算未来 lag 日的日收益率（`ret.shift(-lag)`）
    2. 逐日计算因子值与未来收益的截面 Spearman Rank IC
    3. 对 IC 序列做 t 检验，输出 IC均值、IC_std、t统计量、p值

    **自动给出换仓频率建议：**
    - 最大显著 lag ≤ 5 → 建议周度换仓（`freq='W'`）
    - 最大显著 lag ≤ 20 → 建议月度换仓（`freq='ME'`）
    - 全部不显著 → 因子可能无效
    ######目前默认都是按照周频#####

    ---

    四、换仓日程生成 `build_schedule`

    根据真实交易日历生成换仓三元组列表 `(signal_date, ret_start, ret_end)`：

    **分组规则：**
    - `freq='W'`：按自然周分组，取每周最后一个实际交易日（非固定周五，处理节假日）
    - `freq='ME'`：按自然月分组，取每月最后一个实际交易日
    - `freq='D'`：每个交易日均换仓

    ---

    五、因子分组 `factor_group_from_panel`

    1. 从预计算好的日频因子矩阵中取信号日截面，过滤至有效股票池
    2. 有效股票数 < 50 时跳过该期（防止小样本噪声）
    3. **特殊处理**：若因子值只有 {-1, 1} 两种取值（如 Alpha021），直接映射为 Q1/Q5，跳过分箱
    4. **正常分箱**：对因子值做百分比排名（`rank(pct=True)`），再用 `qcut` 均匀五分组，标签 1-5（1=最低，5=最高）

    使用百分比排名而非原始值分箱，避免因子值分布偏态导致各组股票数不均。

    ---

    六、持仓期收益计算 `calc_adj_returns`
    持仓期收益 = 期末复权收盘价 / 期初复权收盘价 - 1


    期初取 ret_start 当日收盘价（T+1开盘买入的近似）
    - 期末取 ret_end 当日收盘价
    - 返回单行 DataFrame，index = ret_end，columns = 股票代码

    ---

    七、分组收益汇总 `calc_quantile_returns`

    对每个分组（Q1-Q5），取该组内所有股票的持仓期收益均值（等权），返回 `{1,2,3,4,5}` 五列的 DataFrame。

    ---

    八、主回测循环 `run_portfolio_sort`

    ```
    for (signal_date, ret_start, ret_end) in schedule:
        1. 股票池过滤 → 有效股票列表
        2. 因子截面取值 → 五分组
        3. 计算持仓期收益
        4. 各组收益等权平均
        5. 记录因子面板 / 收益面板（用于后续IC计算）

    合并所有期结果 → combined（index=换仓日，columns=Q1~Q5）
    combined['Q5-Q1'] = combined[5] - combined[1]   # 多空组合

    ---

    九、统计检验

    **多空组合 t 检验 `calc_ttest`**

    对 Q5-Q1 多空组合收益序列做单样本 t 检验（H0: 均值=0），输出：
    均值、标准差、年化收益（×52周）、年化夏普（×√52）、t统计量、p值、样本数

    **IC 计算 `calc_ic`**

    逐期计算因子值与当期持仓收益的截面 Spearman Rank IC，汇总输出：
    IC均值、IC标准差、ICIR（IC均值/IC标准差）、IC>0占比、t统计量

    ---

    十、Excel 报告输出

    每个 Alpha 生成独立报告，包含 4 个 Sheet：

    | Sheet | 内容 |
    |-------|------|
    | 统计摘要 | t检验结果 + IC汇总，正负值着色 |
    | IC序列 | IC柱状图（正绿负红）+ 12期滚动均值 + 累积IC |
    | 分组收益 | Q1-Q5 及多空组合持仓期收益分布（直方图+CDF+Sharpe信息框） |
    | 累计净值 | 各分组累计净值走势对比（多空组合用虚线高亮） |

    ---
